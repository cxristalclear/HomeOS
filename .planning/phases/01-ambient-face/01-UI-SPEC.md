# Phase 1 — UI Design Contract: Ambient Face

> Visual and interaction contract for the `/wall` landscape surface.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Governing constraint:** Across-the-room glanceability on a mounted iPad. Every
> size, color, and spacing decision is stress-tested against "readable from several
> feet away in a dim room, landscape orientation."

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — plain Tailwind utility classes throughout |
| Icon library | none — glyphs (✓) used inline as text; no icon library |
| Font | System default (Next.js root layout, no custom font declared) |

**Note:** This project deliberately uses no component library. `components.json`
is absent. shadcn was not initialized and must not be initialized — the existing
phone surface (`/`, `/manage`) is built entirely with Tailwind utility classes and
the wall surface must follow the same pattern. Registry safety gate: not applicable.

---

## Surface Context

The `/wall` route is a **new, distinct surface** — a dark charcoal landscape page
for an always-on, wall-mounted iPad. It is completely separate from the existing
light phone surface (`/`, `/manage`). The phone surface stays untouched.

| Attribute | Phone surface (`/`, `/manage`) | Wall surface (`/wall`) |
|-----------|-------------------------------|------------------------|
| Background | #fafaf9 (warm stone-50) | #1c1917 (stone-950) |
| Orientation | Portrait | Landscape — full viewport |
| Viewing distance | Hand-held | Several feet away |
| Interaction | Touch, tap, scroll | Glance-only in Phase 1 (no interaction) |
| Theme | Light | Dark charcoal |

---

## Spacing Scale

The wall uses the standard Tailwind 4px scale (8-point friendly). No custom tokens.

| Token | px Value | Tailwind Class | Usage |
|-------|----------|---------------|-------|
| xs | 4px | p-1 / gap-1 | Icon gaps, tight inline spacing |
| sm | 8px | p-2 / gap-2 | Owner dot to label, chip internal padding |
| md | 16px | p-4 / gap-4 | Default element spacing, hero internal padding |
| lg | 24px | p-6 / gap-6 | Section separation (hero → "Then today" queue) |
| xl | 32px | p-8 / gap-8 | Layout gutters, page-level horizontal padding |
| 2xl | 48px | p-12 / gap-12 | Major section breaks (top bar → hero) |
| 3xl | 64px | p-16 / gap-16 | Hero vertical padding (enough breathing room for across-the-room read) |

Exceptions:
- Touch targets: not applicable in Phase 1 (display-only, no interactive elements)
- Hero panel: minimum 64px vertical padding (p-16) so text breathes at distance
- Status chips: 12px vertical / 20px horizontal (py-3 px-5) for legibility at arm's length

---

## Typography

The wall's type scale is **substantially larger than the phone** (phone tops out at
text-3xl / 30px). The hero must be readable from several feet away.

| Role | Size (px) | Tailwind Class | Weight | Weight Class | Line Height | Usage |
|------|-----------|---------------|--------|-------------|-------------|-------|
| Body | 16px | text-base | 400 regular | font-normal | 1.5 | Footer disclaimer, chip count labels |
| Label | 20px | text-xl | 600 semibold | font-semibold | 1.4 | "Then today" row task names, top bar app name, section headers |
| Heading | 36px | text-4xl | 600 semibold | font-semibold | 1.2 | Hero owner name, "Then today" section header |
| Display | 72px | text-7xl | 600 semibold | font-semibold | 1.1 | Hero task name (the primary across-the-room read) |

**Tracking:** `tracking-tight` on Heading and Display roles (matches the phone's
established large-heading convention from `page.tsx`).

**Overdue label under the hero task name:** text-2xl (24px), font-normal (400),
text rendered by `overdueLabel()` — always a "when" string ("due today",
"1 day over", "3 days over"), never a debt count. Uses the secondary text color for
the dark surface (stone-400 on dark, see Color section).

**"Then today" queue row sub-detail:** text-sm (14px), font-normal (400), stone-400 —
overdue label per row.

## Color

### Dark wall surface palette

The wall is charcoal-dark. All owner accent colors are tuned to lighter shades than
the phone to meet WCAG AA contrast (≥ 4.5:1 for normal text, ≥ 3:1 for large text
≥ 18px bold / 24px normal) against the dark backgrounds.

| Role | Value | Tailwind Equivalent | Usage |
|------|-------|-------------------|-------|
| Dominant (60%) | #1c1917 | bg-stone-950 | Page background, hero panel base before tint |
| Secondary (30%) | #292524 | bg-stone-900 | Top bar, "Then today" row backgrounds, footer |
| Accent — Christal (sky) | #38bdf8 | text-sky-400 / border-sky-400 | Hero panel tint border, owner dot, owner name text when hero is Christal's |
| Accent — Syd (rose) | #fda4af | text-rose-300 / border-rose-300 | Hero panel tint border, owner dot, owner name text when hero is Syd's |
| Accent — Anyone (neutral) | #a8a29e | text-stone-400 / border-stone-400 | Hero panel tint border, owner dot when owner=anyone |
| Positive / done | #34d399 | text-emerald-400 | Empty state ✓ glyph, "due today" label color |
| Destructive | n/a | n/a | No destructive actions in Phase 1 |
| Primary text | #fafaf9 | text-stone-50 | Hero task name, section headings, chip counts |
| Secondary text | #a8a29e | text-stone-400 | Overdue label, row sub-detail, footer disclaimer |
| Muted text | #57534e | text-stone-600 | De-emphasised elements (not yet used; reserved) |

**Accent reserved for:**
- Hero panel background tint (owner-colored semi-transparent wash, see Hero Panel below)
- Owner identity dots on "Then today" rows
- Owner name text inside the hero panel
- Per-person status chip border and count text

Accent is NOT used on: general typography, the footer, the top bar text, or any
background surface outside the hero panel.

## Hero Panel — Owner Color Wash

The "Next Thing" hero takes the active owner's color as a panel background tint.
This is the primary at-a-glance "whose turn" signal from across the room.

| Owner | Panel background | Border | Owner name color |
|-------|-----------------|--------|-----------------|
| Christal ("me") | bg-sky-950 (deep navy wash) | border-sky-800 | text-sky-400 |
| Syd ("her") | bg-rose-950 (deep rose wash) | border-rose-800 | text-rose-300 |
| Anyone | bg-stone-900 (neutral charcoal) | border-stone-700 | text-stone-400 |

Panel shape: `rounded-3xl border shadow-lg` — consistent with the phone's modal
shape (`rounded-3xl`) and card shape (`rounded-2xl`), scaled up one step for the
larger wall context.

The hero panel fills most of the landscape viewport height (approximately 55-65%
of viewport height) and spans roughly 55% of the viewport width (left column of
a two-column landscape split — hero left, "Then today" + chips right).

## Layout Structure

The `/wall` page is a **full-viewport landscape layout** with no scroll. All
content must fit within the iPad screen (typically 1024×768 on a 10.9" iPad Air,
portrait-mode is disabled for this route).

```
┌──────────────────────────────────────────────────────────────┐
│ TOP BAR (stone-900, h-14)                                    │
│  "Home" wordmark (left)          [no nav in Phase 1]         │
├─────────────────────────────┬────────────────────────────────┤
│                             │                                │
│   HERO PANEL                │  "Then today" queue           │
│   (owner-tinted)            │  (remaining due, worst-first) │
│                             │                                │
│   [owner name]              │  ── section heading ──         │
│   [task name — text-7xl]    │  row  row  row  ...           │
│   [overdue label text-2xl]  │                                │
│                             │  STATUS CHIPS                  │
│                             │  [Christal N]  [Syd N]        │
│                             │                                │
├─────────────────────────────┴────────────────────────────────┤
│ FOOTER (stone-900, h-10)                                     │
│  No-debt disclaimer (center, text-sm, stone-400)             │
└──────────────────────────────────────────────────────────────┘
```

**Column split:** hero col `w-[55%]`, queue+chips col `w-[45%]`.
Both columns use `px-8 py-8` internal padding.
The overall page uses `h-screen overflow-hidden` — no scroll.

## Component Inventory

All components are plain TSX with Tailwind utility classes. No imports from a
component library.

| Component | Location (proposed) | Description |
|-----------|-------------------|-------------|
| `WallPage` | `src/app/wall/page.tsx` | Root client component, `"use client"`, full-viewport layout |
| `WallHero` | `src/app/wall/WallHero.tsx` | Next Thing hero panel — owner tint, big task name, overdue label, or empty state |
| `WallQueue` | `src/app/wall/WallQueue.tsx` | "Then today" list — remaining due items, worst-first rows |
| `WallStatusChips` | `src/app/wall/WallStatusChips.tsx` | Per-person due-today count chips (Christal + Syd) |
| `WallTopBar` | `src/app/wall/WallTopBar.tsx` | Persistent top bar with "Home" wordmark |
| `WallFooter` | `src/app/wall/WallFooter.tsx` | No-debt footer disclaimer |

**Engine functions used (import from `@/lib/engine/`):**
- `bucketTasks` — groups tasks into Today / weekday / Later buckets
- `overdueLabel` — returns the no-debt "N days over" / "due today" / "new" string
- `ownerInView` — used to count Anyone-owned tasks toward both Christal and Syd

**New engine function (WAMB-04):**
- `nextThing(tasks, now)` — returns single worst-first item (or null). Must be
  implemented in `src/lib/engine/` as a pure, side-effect-free function with unit
  tests. Tie-breaking: `since` (most overdue first) → `created_at` → `id`.

## Copywriting Contract

All copy follows the no-debt voice established on the phone surface. Never use
"behind", "overdue by N", "owe", "missed", or any guilt-implying framing.

| Element | Copy | Notes |
|---------|------|-------|
| App wordmark (top bar) | "Home" | Matches existing app title in layout.tsx |
| Hero — owner label | "Christal" / "Syd" / "Anyone" | Uses existing OWNER_NAME map values |
| Hero — overdue label | Output of `overdueLabel(since, now)` | "due today" / "1 day over" / "N days over" / "new" — never a count, never "behind" |
| Hero — empty state glyph | ✓ | Same glyph used on phone empty state |
| Hero — empty state line 1 | "Nothing due." | Source: locked decision — match phone voice |
| Hero — empty state line 2 | "Go do your own thing." | Source: locked decision — match phone voice |
| "Then today" section header | "Then today" | Matches WAMB-05 requirement label |
| "Then today" — empty (when hero is set) | omit section entirely | No rows, no placeholder; hero is sufficient context |
| Status chips — Christal | "Christal · N" where N = due-today count | Plain count, no "tasks" noun — space is limited |
| Status chips — Syd | "Syd · N" where N = due-today count | Same pattern |
| Status chips — zero state | "Christal · 0" (show zero, not hidden) | Showing both chips even at zero confirms the wall is live |
| Footer | "Nothing owed for what slips — start with the one on the left." | No-debt reassurance; directional ("the one on the left") for across-the-room spatial cue |
| Loading state | "Loading…" | text-sm stone-400, centered in hero area |

**Destructive actions:** None in Phase 1. The ambient face is display-only.

## Interaction Contract

Phase 1 is **display-only** — no tappable elements. The wall shows data; all
interaction is deferred to Phase 3 (tap-to-wake) and Phase 4 (room rail actions).

| Interaction | Behavior in Phase 1 |
|-------------|-------------------|
| Tap anywhere | No response (passive display) |
| Swipe | No response |
| Scroll | Prevented (`overflow-hidden` on root) |
| Midnight rollover | `visibilitychange` + timer advance the day buckets (same pattern as phone `page.tsx`) |
| Data refresh | `getRepository().listTasks()` on mount; re-fetched on `visibilitychange` |

**No interactive affordances should be rendered.** No Done buttons, no links, no
toggle. The hero panel and queue rows are non-tappable in this phase.

## States

| State | What renders |
|-------|-------------|
| Loading | Centered "Loading…" in hero area; status chips hidden |
| Empty (nothing due) | Hero shows ✓ + "Nothing due. Go do your own thing." in text-4xl, stone-50; queue section absent; chips still show (both showing 0) |
| Normal — Christal leads | Hero panel bg-sky-950, sky-tinted border; Christal's task at text-7xl; "Then today" lists remaining; chips show both counts |
| Normal — Syd leads | Hero panel bg-rose-950, rose-tinted border; Syd's task at text-7xl |
| Normal — Anyone leads | Hero panel bg-stone-900, neutral border; "Anyone" in text-stone-400 |
| Only one item due | Hero shows it; "Then today" section omitted (nothing to list) |
| Long task name | Task name truncates with ellipsis (`truncate`) at a single line — same pattern as phone; do not wrap to two lines |

## Accessibility

The wall is a passive display rendered on a locked-down iPad in kiosk mode. No
keyboard users. No screen reader in use. Minimum requirements nonetheless:

- `aria-live="polite"` on the hero task name — announces updates when data refreshes
- `role="main"` on the root content area
- Color contrast: all specified accent colors meet WCAG AA for large text (≥ 3:1)
  against their respective panel backgrounds:
  - sky-400 (#38bdf8) on stone-950 (#1c1917): ~8.9:1 — passes AA normal and large
  - rose-300 (#fda4af) on stone-950 (#1c1917): ~7.1:1 — passes AA normal and large
  - emerald-400 (#34d399) on stone-950 (#1c1917): ~8.3:1 — passes AA normal and large
  - stone-50 (#fafaf9) on stone-950 (#1c1917): ~19.3:1 — passes AAA
- `lang="en"` on `<html>` (already set in `layout.tsx`)
